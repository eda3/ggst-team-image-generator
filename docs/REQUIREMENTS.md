# 機能要件・非機能要件

## 1. 機能要件

### 1.1 Phase 1: 基本機能（CLIツール）

#### 1.1.1 チーム画像生成
- **入力**
  - チーム名（文字列）
  - メンバー情報（3名分）
    - プレイヤー名
    - 使用キャラクター（キャラクターID or 名前）
- **出力**
  - PNG画像ファイル（指定ディレクトリに保存）

#### 1.1.2 キャラクター管理
- GGSTの全キャラクターに対応
- キャラクター画像（アイコン/ポートレート）の管理
- キャラクター名のエイリアス対応（例: "sol" → "Sol Badguy"）

#### 1.1.3 レイアウトオプション
- 横並び（3キャラ横一列）
- 縦並び（3キャラ縦一列）
- カスタムサイズ指定

### 1.2 Phase 2: 拡張機能

#### 1.2.1 一括生成
- JSON/YAML形式でチームリストを読み込み
- 複数チームの画像を一括生成

#### 1.2.2 テンプレートシステム
- 複数のデザインテンプレート
- カスタムテンプレートの追加対応

### 1.3 Phase 3: Webツール化

#### 1.3.1 ブラウザUI
- チーム情報の入力フォーム
- キャラクター選択（ドロップダウン or アイコンクリック）
- リアルタイムプレビュー
- ダウンロードボタン

#### 1.3.2 追加機能
- 生成履歴の保持（LocalStorage）
- 共有用URL生成

### 1.4 Phase 4: 対戦カード生成

- 2チームを左右に配置した対戦カード画像
- VS表示
- 大会名・ラウンド情報の追加

---

## 2. 非機能要件

### 2.1 パフォーマンス
- 単一画像生成: 500ms以内（Bunの高速性を活かす）
- 一括生成（10チーム）: 3秒以内

### 2.2 拡張性
- 新キャラクター追加が容易な設計
- テンプレート追加が容易な設計
- CLI → Web への移行を考慮した共通コア設計

### 2.3 保守性
- TypeScriptによる型安全性
- モジュール分離による責務の明確化
- テストコードの整備（Bun test）
